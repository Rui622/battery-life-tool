产品续航分析方法说明

1. 简介与测试流程概述

1.1. 测试流程概述

本文档中的所有续航数据，均基于一套标准化的“功耗建模”测试流程。该流程通过将设备的复杂用电行为分解为独立的、可测量的单元，从而实现对续航时间的精确理论计算。具体测试流程如下：

  • 功耗单元分解与测量:
    • 事件测量: 使用功率分析仪等专业仪器，精确捕捉并测量每一个独立操作（如电机动作、屏幕点亮、网络收发）在发生时的瞬时电流（mA）与持续时间（秒）。
    • 状态测量: 测量设备在长时间维持某个状态（尤其是深度休眠）下的静态电流（μA）。
  • 建立使用模型:

    根据产品的典型应用场景，定义一个标准化的24小时使用模型。此模型规定了每个独立操作在一天内发生的预估频率（次/天）。

  • 计算每日总耗电量:

    将步骤1中测得的各功耗单元数据，代入步骤2中定义的使用模型，通过本文后续章节详述的功耗公式，计算出设备在特定场景下的每日平均总耗电量（mAh/天）。

  • 计算理论续航:

    最后，用电池的实际可用电量除以计算出的每日总耗电量，得出最终的理论续航天数。

1.2. 目的

本文旨在建立并说明一套标准化的产品续航时间分析方法。通过定义清晰的计算原理、公式及应用模型，为产品研发、市场、销售等各部门提供一个统一、客观的参考框架，以科学地评估和沟通产品续航性能。

1.3. 适用范围

本方法适用于所有需要进行理论续航评估的电池供电型产品。

2. 续航计算的核心原理与公式

2.1. 基本原理

产品理论续航时间由两大核心要素决定：电池的可用能量 与 设备的能量消耗速率。

基本计算公式：

理论续航天数 = 电池总可用电量 (mAh) / 设备每日平均耗电量 (mAh/天)

2.2. 核心公式详解

2.2.1. 电池总可用电量 (Usable Battery Capacity)

该值指电池在保证安全和寿命的前提下，实际可用于驱动设备的总电量。

计算公式:

总可用电量 = 电池标称容量 × 可用容量百分比
  • 电池标称容量 (mAh): 电池出厂时标注的最大电量。
    • 采用多规格容量的原因 (如 3200 / 2800 / 1800mAh): 在分析中涵盖多种电池规格，旨在量化不同等级的电池容量对产品续航性能的直接影响。这为评估不同产品版本（如标准版、长续航版）的性能表现提供了数据基准，同时也为终端用户在购买成本与更换周期之间进行权衡提供了客观的参考依据。
  • 可用容量百分比 (%): 一个基于安全和可靠性设定的工程系数。设定此系数是为了防止因电池电压低于设备“截止电压”而导致的功能异常，并能有效延长电池的整体使用寿命。在本次分析的所有计算中，统一采用 79.1% 作为“可用容量百分比”。

2.2.2. 设备每日平均耗电量 (Average Daily Consumption)

该值指设备在24小时内所有操作消耗电量的总和，由两类功耗构成：

  • A. 短时事件功耗 (Event-Driven Consumption): 设备在一天内多次执行的、持续时间较短的瞬时动作（如亮屏、电机转动、网络通信等）。

    计算公式:

    事件功耗 (mAh) = 事件电流(mA) × (单次时长(秒)/3600) × 每日次数
  • B. 持续状态功耗 (State-Based Consumption): 设备长时间维持在某种状态下的消耗（主要是休眠/待机）。

    计算公式:

    状态功耗 (mAh) = 状态电流(mA) × 每日持续小时数(h)

因此，每日总耗电量的完整计算公式为：

每日总耗电量 = Σ (所有短时事件功耗) + Σ (所有持续状态功耗)

(注：Σ为求和符号)

3. 计算模型的应用

为了模拟不同的用户使用习惯，我们将计算方法应用于以下两种标准模型。

3.1. 模型A：连续工作模型

模型定义: 模拟设备全年处于高频使用状态的场景。

计算流程:

  • 场景一：不联网状态
    1. 计算各部分日耗电量:
      • A. 短时事件功耗总和
        • 事件1 (电机动作): 140mA × (5秒 / 3600) × 10次 ≈ 1.944 mAh
        • 事件2 (屏幕点亮): 28mA × (10秒 / 3600) × 20次 ≈ 1.556 mAh
        • 事件功耗总和 = 1.944 + 1.556 = 3.500 mAh
      • B. 持续状态功耗总和
        • 举例 (纯静态休眠): 0.01mA × 24小时 = 0.24 mAh
    2. 求和得出每日总耗电量:
      • 3.500 mAh (事件) + 0.24 mAh (状态) = 3.74 mAh/天
    3. 计算最终续航 (以3200mAh电池为例):
      • 总可用电量 = 3200mAh × 79.1% = 2531.2 mAh
      • 续航天数 = 2531.2 mAh / 3.74 mAh/天 ≈ 677 天
  • 场景二：联网状态
    1. 方法说明: 对于联网设备，其待机功耗不仅包含纯静态休眠，还包含了为维持连接而进行的周期性网络活动（如心跳、数据上报/下发等）。为精确计算，通常采用功率分析仪对设备进行长时间（如24小时）测量，仪器会自动计算出包含所有后台网络活动的 “等效平均待机电流”。
    2. 计算各部分日耗电量:
      • A. 短时事件功耗总和
        • 本地操作事件功耗: 与场景一相同，为 3.500 mAh。
      • B. 持续状态功耗总和 (联网平均待机)
        • 举例: 假设通过功率分析仪测得的“等效平均待机电流”为 228.5 μA (即 0.2285 mA)。
        • 联网待机功耗 = 0.2285mA × 24小时 ≈ 5.484 mAh
    3. 求和得出每日总耗电量:
      • 3.500 mAh (事件) + 5.484 mAh (联网待机) = 8.984 mAh/天
    4. 计算最终续航 (以3200mAh电池为例):
      • 总可用电量 = 2531.2 mAh
      • 续航天数 = 2531.2 mAh / 8.984 mAh/天 ≈ 282 天

3.2. 模型B：季节性工作模型 (以联网状态为例)

  • 3.2.1. 模型背景与定义
    • 背景说明: 引入此模型的目的是为了更真实地模拟具有明显使用周期的产品。例如，仅在冬季供暖季工作的温控阀类产品，其大部分时间的功耗极低。相比于全年高频工作的“连续工作模型”，季节性模型能更准确地评估这类产品在实际应用场景中的续航表现。
    • 模式定义: 模型通过“工作月数 + 休眠月数”的形式来定义一年的工作周期。例如：
      • 4+8模式: 代表设备一年中有4个月处于高频工作的“工作期”，其余8个月处于极低功耗的“休眠期”。
      • 6+6模式: 代表6个月工作期与6个月休眠期。
      • 8+4模式: 代表8个月工作期与4个月休眠期。
  • 3.2.2. 计算流程
    1. 定义“工作期”与“休眠期”的日耗电量:
      • 工作期日耗电量 (C_active): 采用“模型A-场景二：联网状态”的计算结果。
        • 举例: C_active = 8.984 mAh/天。
      • 休眠期日耗电量 (C_sleep): 此期间设备通常断开网络连接，仅处于纯静态休眠状态。
        • 举例: 0.01mA × 24小时 = 0.24 mAh/天。
    2. 计算“年度平均每日耗电量”:
      • 第一步：计算年度总耗电量

        计算公式:

        年度总耗电量 = (Cactive × 工作期天数) + (Csleep × 休眠期天数)

        参数说明: 工作/休眠期天数 通常由月份定义，可通过 月数 × 30.4 (一年中平均每月的天数) 估算。

        计算举例 (以“8个月工作，4个月休眠”为例):
        年度总耗电量 = (8.984 × (8 × 30.4)) + (0.24 × (4 × 30.4)) ≈ 2185 + 29 = 2214 mAh/年

      • 第二步：计算年度平均日耗电量

        计算公式:

        年度平均日耗电量 = 年度总耗电量 / 365

        计算举例: 2214 mAh/年 / 365天 ≈ 6.07 mAh/天

    3. 计算最终续航 (以3200mAh电池为例):
      • 总可用电量: 2531.2 mAh
      • 续航天数: 2531.2 mAh / 6.07 mAh/天 ≈ 417 天

4. 结论解读与关键前提

4.1. 计算结果的定位与价值

  • 每日耗电量 (KPI): 作为衡量产品功耗性能的核心技术指标，该数值为评估固件优化效果、对比不同工作模式的节能效益提供了客观、量化的依据。
  • 理论续航天数 (Benchmark): 作为在特定条件下得出的性能基准，其主要价值在于为产品规格定义、成本效益分析和市场沟通提供统一的数据支持，并帮助管理用户预期。

4.2. 关键前提（计算依赖项）

所有理论计算结果的有效性，均严格依赖于以下前提条件。对任何前提条件的变更都将直接影响最终计算结果。

  • 可用容量的设定: 续航结果由章节2.2.1中定义的“可用容量百分比”直接决定。
  • 操作频率的定义: 续航结果与章节2.2.2中定义的每日操作次数强相关。
  • 标准测试环境: 功耗参数基于标准化的温度和实验室环境测得，实际环境的差异可能导致结果偏差。
  • 特定的软硬件版本: 计算结果与测试时所用的特定固件和硬件版本绑定。
